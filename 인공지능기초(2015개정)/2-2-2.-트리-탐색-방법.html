인공지능의 길 찾기, '탐색(Search)'

여러분, 체스나 바둑을 둘 때 인공지능은 어떻게 다음 수를 결정할까요? 가능한 모든 경우의 수를 그려보면, 그 모양이 마치 나뭇가지가 계속해서 뻗어나가는 '트리(Tree)' 모양이 됩니다.

하지만 바둑처럼 경우의 수가 우주에 있는 별의 수만큼 많다면 어떨까요? 모든 나뭇가지(트리)를 끝까지 다 확인하는 것은 슈퍼컴퓨터로도 불가능합니다.

💡 탐색(Search)이란?

이처럼 수많은 경우의 수(트리) 속에서 규칙을 지키며 '정답(목표 상태)에 도달하는 경로를 효과적으로 찾아내는 과정'을 탐색이라고 합니다.

두 가지 주요 트리 탐색 방법 ✌️

방법 1

무정보 탐색
(Uninformed Search)

문제에 대한 특별한 단서나 힌트 없이, 정해진 규칙에 따라 모든 길을 샅샅이 뒤져보는 방법입니다.

  • 비유: 운동장에서 보물찾기를 할 때, 단서 없이 구석구석 모든 모래를 다 파보는 것과 같습니다.
  • 특징: 시간이 오래 걸리지만, 정답이 있다면 언젠가는 반드시 찾아냅니다.
방법 2

휴리스틱 탐색
(Heuristic Search)

문제 속에 숨겨진 힌트(정보)를 활용해서, 정답 확률이 높은 곳만 골라 똑똑하게 탐색하는 방법입니다.

  • 비유: "보물은 큰 나무 근처에 있다"는 힌트를 듣고, 나무 주변만 집중적으로 파보는 것입니다.
  • 특징: 똑똑하게 일부만 찾아보기 때문에 시간을 획기적으로 아낄 수 있습니다.

무정보 탐색 (Uninformed Search)

상태 공간에 내재된 정보와 상관없이 정해진 규칙에 따라 모든 상태를 맹목적으로 탐색하는 방법입니다. 경우의 수가 많을 때는 탐색 시간이 오래 걸릴 수 있습니다.

깊이 우선 탐색 (DFS)

루트 노드에서 시작하여 한 방향(분기)으로 갈 수 있을 때까지 가장 깊은 곳까지 탐색한 후, 더 이상 갈 곳이 없으면 되돌아와서 다른 방향을 탐색하는 방법입니다.

A 1번째
B 2번째
D 3번째
E 4번째
C 5번째
F 6번째
G 7번째

너비 우선 탐색 (BFS)

루트 노드에서 시작하여 인접한 노드를 먼저 탐색하는 방법입니다. 같은 깊이(레벨)의 노드를 모두 탐색한 후, 다음 레벨로 넘어갑니다.

A 1번째
B 2번째
D 4번째
E 5번째
C 3번째
F 6번째
G 7번째

휴리스틱 탐색 (Heuristic Search)

상태 공간에 내재된 정보(휴리스틱 정보)를 활용하여 트리의 일부만 효과적으로 탐색하는 방법입니다. 정보 있는 탐색이라고도 부르며, 경우의 수가 너무 많아 무정보 탐색으로 해결하기 어려운 문제에 적합합니다.

휴리스틱(Heuristic)이란?

모든 경우를 무작정 다 살펴보는 대신, 인간의 직관이나 경험을 바탕으로 가장 정답에 가까울 것 같은 방향을 추정하여 우선적으로 시도하는 방법입니다.

예시: 내비게이션으로 경로를 찾을 때, 현재 위치에서 사방의 모든 도로를 탐색하지 않고 목적지가 위치한 방향(직선거리)으로 뻗은 도로를 먼저 탐색하는 것이 휴리스틱 정보를 활용한 좋은 예입니다.

주요 트리 탐색 방법

무정보 탐색 알고리즘

깊이 우선 탐색 (DFS)

Depth-First Search

개념: 갈 수 있는 한 끝까지 '한 우물만 깊게' 파고들어 탐색하는 방식입니다. 막다른 길에 다다르면 가장 최근의 갈림길로 되돌아옵니다.

예시: 미로 찾기에서 오른손으로 벽을 짚고 끝까지 걸어가 보는 것과 같습니다.

특징: 기억해야 할 갈림길이 적어 메모리를 적게 쓰지만, 가장 빠른 지름길을 찾는다는 보장이 없습니다.

# DFS 핵심 동작
def dfs(graph, node, visited):
    if node not in visited:
        visited.append(node)       # 현재 위치 방문
        for next_node in graph[node]:
            dfs(graph, next_node, visited) # 더 깊이 파고들기!

너비 우선 탐색 (BFS)

Breadth-First Search

개념: 시작점에서 가까운 주변부터 '얕고 넓게' 모두 훑어보며 점차 탐색 범위를 넓혀가는 방식입니다.

예시: 잔잔한 호수에 돌을 던졌을 때, 물결이 동심원을 그리며 서서히 퍼져나가는 모습과 같습니다.

특징: 반드시 '가장 빠른 지름길(최단 경로)'을 찾아내지만, 찾아본 모든 곳을 기억해야 해서 메모리가 많이 필요합니다.

# BFS 핵심 동작
def bfs(graph, start):
    queue = [start] # 큐(대기열) 사용
    while queue:
        node = queue.pop(0) # 가장 먼저 온 것부터 확인
        # 주변의 모든 노드를 대기열에 추가!

휴리스틱 탐색 알고리즘

언덕 등반 탐색 (Hill Climbing)

개념: 눈을 가리고 산을 오를 때, 주변을 더듬어 보고 무조건 '지금보다 더 높은 곳'으로만 한 걸음씩 이동하는 방식입니다.

특징: 현재 상태에서 당장 좋아 보이는 방향으로만 움직이기 때문에 계산이 매우 빠릅니다.

치명적인 함정: 지역 최적해(Local Maxima)

동네 뒷산 꼭대기에 올랐을 뿐인데, 사방이 내리막길이라 "여기가 에베레스트 정상(최고의 정답)이구나!" 하고 착각하여 탐색을 멈춰버리는 문제가 발생할 수 있습니다.

최고 우선 탐색 (Best-First Search)

개념: 목적지까지 '앞으로 얼마나 남았을까?' (예상 남은 거리)만 평가해서, 가장 목적지와 가까워 보이는 길을 최우선으로 선택합니다.

특징: 목적지를 향해 빠르게 직진하려는 성향이 강합니다. 하지만 앞에 커다란 장애물이 있다면 직진하다가 오히려 더 먼 길을 돌아가게 될 수도 있습니다.

A* (에이스타) 알고리즘

개념: 가장 똑똑하고 널리 쓰이는 길 찾기 알고리즘입니다. '지금까지 걸어온 실제 거리'와 '앞으로 남은 예상 거리'를 모두 더해서 가장 유리한 길을 선택합니다.

예시: 자동차 내비게이션이나 게임 캐릭터의 길 찾기에 주로 사용됩니다.

# A* 알고리즘의 핵심 판단 공식
# f(n) = g(n) + h(n)

# g(n): 출발점에서 현재 위치까지 온 '실제' 비용(거리)
# h(n): 현재 위치에서 목적지까지의 '예상' 비용 (휴리스틱)
# f(n): 이 두 값을 더한 총 비용. A*는 항상 f(n)이 가장 작은 길을 선택합니다!

탐색으로 퍼즐 풀기: 강 건너기 문제

탐색 방법이 실제로 어떻게 쓰이는지 유명한 논리 퍼즐을 통해 알아볼까요?

🧑‍🌾 농부의 강 건너기 대작전

농부가 늑대, 양, 양배추를 배에 싣고 무사히 강을 건너야 합니다.

🚨 반드시 지켜야 할 규칙

  • 배가 작아서 농부는 한 번에 한 가지만 배에 실을 수 있습니다.
  • 농부가 없을 때, 늑대와 양만 남겨두면 늑대가 양을 잡아먹습니다!
  • 농부가 없을 때, 양과 양배추만 남겨두면 양이 양배추를 먹어버립니다!

인공지능은 이 문제를 풀기 위해 현재 상태(모두 출발지에 있음)에서 배를 움직이는 모든 경우의 수를 트리 모양으로 그려나갑니다.
탐색 도중 규칙을 어기는 경로(예: 늑대와 양만 남는 경우)는 바로 잘라내버리고, 모두 무사히 건너간 상태(목표 상태)에 도달할 때까지 끊임없이 탐색을 진행하여 정답 경로를 찾아냅니다.

Q. 인공지능은 세상의 모든 문제를 다 '탐색'으로 해결할 수 있나요?

A. 아니요! 탐색으로 풀 수 없는 문제도 아주 많습니다.

탐색을 하려면 문제를 [초기 상태 ➔ 여러 가지 이동 규칙(상태 공간) ➔ 목표 상태]로 확실하게 정의할 수 있어야 합니다.

예를 들어, "태양계의 행성을 모두 말해줘!" 같은 문제는 길을 찾는 것이 아니라 '사실'을 알고 있어야 하는 문제입니다. 이런 경우에는 탐색이 아니라 지식을 저장하고 '추론'하는 전혀 다른 인공지능 기술이 필요하답니다.
